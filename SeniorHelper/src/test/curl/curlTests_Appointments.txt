Phase 2 - SeniorHelper Appointments API Testing
------------------------------------------------

1. Create a valid appointment

curl -s -X POST http://localhost:8080/api/appointments ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"Dentist cleaning\",\"notes\":\"Bring ID\",\"start\":\"2025-09-10T09:00:00\",\"end\":\"2025-09-10T09:30:00\",\"location\":\"Aspen Dental\"}"

Result:
{"id":2004,"title":"Dentist cleaning","notes":"Bring ID","start":"2025-09-10T09:00:00","end":"2025-09-10T09:30:00","location":"Aspen Dental"}
***********************************************************

2. Create an appointment with bad data (expect 400)

curl -s -X POST http://localhost:8080/api/appointments ^
  -H "Content-Type: application/json" ^
  -d "{\"title\":\"\",\"start\":\"2000-01-01T09:00:00\",\"end\":\"2000-01-01T09:30:00\"}"

Result:
{"path":"/api/appointments","message":"end: must be a date in the present or in the future","status":400,"timestamp":"2025-09-05T16:11:08.878731100Z"}
