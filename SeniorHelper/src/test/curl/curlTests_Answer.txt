Answer Feature cURL Tests
***

Happy Paths (Module, Lesson, Quiz AND Question must exist before performing CRUD on 'Answer')

***
1. Create a new answer.                                     -> EXPECT 201
curl -i -X POST http://localhost:8080/api/modules/1/lessons/1/quizzes/1/questions/1/answers -H "Content-Type: application/json" -d @SeniorHelper/src/test/curl/answers.json

RESULT: HTTP/1.1 201
    Content-Type:       application/json
    Transfer-Encoding:  chunked
***
2. Get all answers.                                         -> EXPECT 200
curl -i -X GET http://localhost:8080/api/modules/1/lessons/1/quizzes/1/questions/1/answers

RESULT: HTTP/1.1 200
    Content-Type:       application/json
    Transfer-Encoding:  chunked
***
3. Get one answer.                                          -> EXPECT 200
curl -i -X GET http://localhost:8080/api/modules/1/lessons/1/quizzes/1/questions/1/answers/1

RESULT: HTTP/1.1 200
    Content-Type:       application/json
    Transfer-Encoding:  chunked
***
4. Update an answer.                                        -> EXPECT 200
curl -i -X PUT http://localhost:8080/api/modules/1/lessons/1/quizzes/1/questions/1/answers/1 -H "Content-Type: application/json" -d "{\"text\": \"Updated answer\", \"correct\": false}"

RESULT: HTTP/1.1 200
    Content-Type:       application/json
    Transfer-Encoding:  chunked
***
5. Delete an answer.                                        -> EXPECT 204
curl -i -X DELETE http://localhost:8080/api/modules/1/lessons/1/quizzes/1/questions/1/answers/1

RESULT: HTTP/1.1 204
***
