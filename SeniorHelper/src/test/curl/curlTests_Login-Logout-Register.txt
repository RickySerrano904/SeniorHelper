Login/Logout/Register Feature Tests

***********************************************************
EDGE CASES
***********************************************************

1. Login with missing fields:
curl -i -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{}"

Result:
HTTP/1.1 400
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
Connection: close
{"path":"/api/auth/login","message":"password: is required","status":400,"timestamp":"2025-09-27T23:00:12.380535200Z"}

***********************************************************

2. Login with non-existent username:
curl -i -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"nope\",\"password\":\"x\"}"

Result:
HTTP/1.1 404
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
{"path":"/api/auth/login","message":"User not found","status":404,"timestamp":"2025-09-27T23:03:06.924512900Z"}

***********************************************************

3. Login with wrong password:
curl -i -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"caregiver1\",\"password\":\"WRONG\"}"

Result:
HTTP/1.1 401
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
{"path":"/api/auth/login","message":"Invalid credentials","status":401,"timestamp":"2025-09-06T19:59:04.732902700Z"}

***********************************************************
HAPPY PATH
***********************************************************

4. Register a new user(auto login, and returns a token):
curl -i -s -X POST http://localhost:8080/api/auth/register -H "Content-Type: application/json" -d "{\"username\":\"alice\",\"password\":\"letmein\"}"

Result:
HTTP/1.1 201
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
{"token":"<TOKEN>","expiresAt":"<DATETIME>","user":{"id":1,"username":"alice","role":null}}

***********************************************************

5. Logout (requires token in header):
curl -i -s -X POST http://localhost:8080/api/auth/logout -H "X-AUTH-TOKEN: <TOKEN>"

Result:
HTTP/1.1 200
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
{"message":"Logged out successfully"}

***********************************************************

6. Login with existing user (returns new token):
curl -i -s -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"alice\",\"password\":\"letmein\"}"

Result:
HTTP/1.1 200
Content-Type: application/json
Transfer-Encoding: chunked
Date: <DATETIME>
{"token":"<TOKEN>","expiresAt":"<DATETIME>","user":{"id":1,"username":"alice","role":null}}
